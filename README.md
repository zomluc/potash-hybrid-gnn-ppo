# Hybrid GNN-PPO for Import Potash Supply-Chain Resilience

This repository contains the simulation environment, reinforcement-learning models, stochastic-programming baselines, and experiment scripts used for the revised manuscript on resilience planning in an import potash fertilizer supply chain.

The code implements a weekly simulation environment with overseas supply channels, Chinese receiving gateways, stochastic demand, route disruptions, supplier disruptions, price fluctuations, inventory dynamics, backlog, backup-route reservation, transport reserve, and futures hedging.

## Repository Contents

- `supply_chain_environment.py`, `supply_chain_dynamics.py`, `simulation_types.py`: supply-chain environment and weekly transition dynamics.
- `rl_environment.py`: Gymnasium-compatible wrapper and observation/action encoding.
- `rl_models.py`: Plain PPO, Transformer-PPO, GAT-PPO, GNN-PPO, and Hybrid GNN-PPO model definitions.
- `ppo_trainer.py`: unified PPO training loop, warm-start procedure, checkpointing, and training-curve output.
- `two_stage_sp.py`, `scenario_tree.py`, `decision_strategy.py`: stochastic planning and rolling two-stage stochastic-programming baselines.
- `compare_methods.py`: common-seed method comparison with paired statistical tests.
- `run_sensitivity_analysis.py`: one-factor and joint supply-transport stress tests.
- `analyze_policy_behavior.py`: policy-behavior trajectory analysis.
- `results.zip` or `results/`: machine-readable CSV/JSON outputs and paper figures from the reported experiments. If the results are provided as `results.zip`, unzip it in the repository root before inspecting reproduced outputs or rerunning post-training analyses.

Large PyTorch checkpoint files (`*.pt`) are not included in this lightweight repository package. They can be regenerated by running the training scripts below. If only Python source files and `results.zip` are uploaded through the GitHub web interface, the repository remains usable as long as `README.md`, `requirements.txt`, all Python scripts, and the results archive are present.

## Environment

The experiments were run with Python, PyTorch, NumPy, SciPy, pandas, matplotlib, and Gymnasium. A CPU-only run is sufficient, although training the graph-based PPO variants may take several hours.

Install dependencies with:

```bash
pip install -r requirements.txt
```

## Training

Run the five PPO training entrypoints from the repository root:

```bash
python train_plain_ppo.py
python train_transformer_ppo.py
python train_gat_ppo.py
python train_gnn_ppo.py
python train_hybrid_ppo.py
```

The default training configuration uses 2,000 episodes, imitation warm-start, evaluation every 80 episodes, and the common training seed `20260308`. The best validation checkpoint for each method is saved under `results/<method>/ppo_best.pt`.

## Main Comparison

After training the PPO methods, run:

```bash
python compare_methods.py
```

This evaluates stochastic planning, rolling two-stage stochastic programming, Plain PPO, Transformer-PPO, GAT-PPO, GNN-PPO, and Hybrid GNN-PPO on the common 20 test seeds:

```text
101, 202, 303, 404, 505, 606, 707, 808, 909, 1001,
1102, 1203, 1304, 1405, 1506, 1607, 1708, 1809, 1901, 2002
```

The comparison outputs are written to `results/comparison/`, including per-seed episode metrics and pairwise statistical tests. If the uploaded repository contains `results.zip`, first extract it so that `results/comparison/` is available.

## Sensitivity and Policy-Behavior Analyses

Run the sensitivity and joint stress tests with:

```bash
python run_sensitivity_analysis.py
```

Run the policy-behavior analysis with:

```bash
python analyze_policy_behavior.py
```

These scripts reuse the trained PPO checkpoints and write CSV/JSON summaries and figures under `results/sensitivity/` and `results/policy_behavior/`. The archived reported outputs can be inspected after extracting `results.zip`.

## Reproducibility Notes

- PPO training seed: `20260308`.
- PPO validation seeds: `1101, 1202, 1303, 1404, 1505`.
- Imitation warm-start seeds: `2101, 2202, 2303, 2404, 2505, 2606, 2707, 2808`.
- Main comparison seeds: listed above.
- Sensitivity-analysis seeds: `611, 722, 833, 944, 1055`.

The environment and experiment scripts are deterministic conditional on the specified seeds. Minor numerical differences may occur across PyTorch, NumPy, and hardware versions.
